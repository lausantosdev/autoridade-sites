## 2. CI: GitHub Actions

**Problema:** `make test` roda local, mas nada valida no push/PR.

**Criar:** `.github/workflows/test.yml`

```yaml
name: Tests

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - name: Install dependencies
        run: pip install -r requirements.txt -r requirements-dev.txt
      - name: Run tests
        run: python -m pytest tests/ -v
```

## 4. `pyproject.toml`

**Problema:** Projeto sem metadados formais. `requirements.txt` solto não declara nome, versão, autor, nem Python mínimo.

**Criar:** `pyproject.toml` na raiz

```toml
[project]
name = "sitegen"
version = "1.0.0"
description = "Gerador de sites SEO para negócios locais brasileiros"
requires-python = ">=3.11"
dependencies = [
    "fastapi",
    "uvicorn[standard]",
    "websockets",
    "pyyaml",
    "python-dotenv",
    "openai",
    "google-generativeai",
    "tqdm",
    "pillow",
]

[project.optional-dependencies]
dev = [
    "pytest>=8.0.0",
    "pytest-cov>=5.0.0",
]

[tool.pytest.ini_options]
testpaths = ["tests"]

[tool.coverage.run]
source = ["core"]
omit = ["tests/*"]
```

> Após criar o `pyproject.toml`, o `Makefile` pode usar `pip install -e .[dev]`
> em vez de instalar `requirements.txt` + `requirements-dev.txt` separados.
> Atualizar o target `install-dev` no Makefile.
